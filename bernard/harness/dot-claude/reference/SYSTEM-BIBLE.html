0 · Overview — three surfaces, one record

The system is a private personal life-wiki for one curator, with a sanitization front-door and a read-only WhatsApp companion as a second audience.

① Sanitizer pre-wiki

~/Devops/life-wiki-sanitizer

Raw documents in → redacted, classified, pseudonymized → dropped into the wiki's ingest/. Models here are firewalled from live data.

② Wiki source of truth

~/life-wiki

Markdown pages organized by domain. GBrain indexes them for search; Claude Code edits them. Local git only — no remote.

③ Companion (Bernard) read-only

VPS your-vps

A scoped, one-way mirror of the health wiki answers one allowlisted person over WhatsApp. Zero public surface.

Mental model: GBrain is the search engine; Claude Code (or Codex/Gemini) is the editor; the .md files are the database they both operate on. The sanitizer guards the front door; the companion is a guarded read-only window.

4 · Process flows

4.1 Ingest — raw document → cited wiki page

sequenceDiagram
  autonumber
  actor C as Curator
  participant S as Source
  participant SAN as Sanitizer
  participant ING as ingest/
  participant CC as Claude Code
  participant L as wiki-lint
  participant V as post-ingest-verify
  participant GB as GBrain
  S->>SAN: raw document
  SAN->>SAN: redact (targets), classify, pseudonymize
  SAN->>ING: .md + .meta.json (pre-sanitized)
  C->>CC: /ingest
  CC->>L: structural lint (pre-flight)
  L-->>CC: 0 critical (else blocked)
  CC->>CC: scan for prompt injection
  CC->>CC: extract entities, build wikilinks, cite claims
  CC->>ING: read + route to wikis/** (move .md + sidecar together)
  CC->>V: PHI tripwire (230 patterns)
  V-->>CC: clean
  CC->>GB: import --no-embed → embed --stale → extract links
  CC->>C: update sources.md, log.md, AGENT-HANDOFF.md
  • Hard rule: moving a file to raw/ is not a complete ingest — pages, citations, links, indexes, logs, and the handoff must all be updated.
  • Gate: the git pre-commit hook re-checks PHI / audio / sidecars before anything enters history.

4.3 Inbox loop — closing the feedback gap

flowchart LR
  Q[Bernard logs gap line on VPS] --> SY[mirror sync: atomic mv + dedupe]
  SY --> LOC[.claude/inbox/wiki-question-inbox.md]
  SY --> ARC[(VPS archive)]
  LOC --> TR[/inbox-triage skill/]
  TR --> FIX[create/enrich page · or file under shared/questions]
  FIX --> MV[move line → inbox-archive.md done date→page]
  FIX --> GBR[GBrain reindex]
  GBR --> NEXT[next mirror ships fix → Bernard can now answer]

Dedupe is against the inbox and the archive, so triaged lines never resurrect on the next pull. The remote file is archived-and-truncated after each successful pull (orphan-recovered if the archive step fails).

5 · Security model — defense in depth

Ten layers, ordered from front door to external surface. The principle: sanitize once, then make every later stage fail closed.

classDiagram
  class Sanitization {
    stage: pre-wiki
    +redact(targets)
    +pseudonymize(PATIENT-001, MOM-001)
    +classify + sidecar
  }
  class GuardSensitive {
    repo: sanitizer · PreToolUse
    +denyRead(landing, vault, ingest, ~life-wiki)
    +selfProtect(.claude .codex .cursor .gemini)
  }
  class GuardWiki {
    repo: life-wiki · PreToolUse
    +ask(ingest/, raw/)
    +deny(audio .m4a .mp3 ...)
    +selfProtect(hooks, settings)
  }
  class PreCommitGate {
    scope: ALL committers
    +blockAudio(ext + magic)
    +phiTripwire(encoded variants)
    +sidecarPairing()
  }
  class PostIngestVerify {
    scope: batch close-out
    +scan(230 patterns)
    +noDuplicates() + noOrphans()
  }
  class NightlyLeakScan {
    schedule: 02:30
    +scan5tier()
    +diffBaseline()
    +writeStatus(clean | DEFINITE)
  }
  class MirrorGate {
    +requireFreshClean(<48h)
    +failClosed()
    +scopeAllowlist(health, shared, journal/entries)
  }
  class VPSHardening {
    +allowlistWhatsApp(one number)
    +zeroPublicSurface(PDF attach, no Funnel)
    +toolsElevated(false) + tailnet/loopback only
  }

  GuardSensitive --> GuardWiki
  GuardWiki --> PreCommitGate
  PreCommitGate --> PostIngestVerify
  PostIngestVerify --> NightlyLeakScan
  NightlyLeakScan --> MirrorGate
  MirrorGate --> VPSHardening

Why each later stage repeats the PHI check: the 2026-06-11 incident (patient name copied into 147 pages from "sanitized" sources via encoded forms) proved a single upstream check isn't enough. Layers 5–8 each re-verify with the encoded-variant battery, and the mirror won't publish unless layer 7 says clean.

6 · Lifecycles

Page status

stateDiagram-v2
  [*] --> staged
  staged --> active: cited + linked + indexed + approved
  active --> archived: superseded but preserved
  staged --> archived: abandoned
  archived --> [*]

A health page with no source_paths can never be active.

Leak-scan gate

stateDiagram-v2
  [*] --> clean
  clean --> DEFINITE: new findings beyond baseline
  DEFINITE --> clean: triage + --accept-baseline
  clean --> stale: status >48h old
  stale --> clean: re-run scan
  note right of DEFINITE: mirror push REFUSED
  note right of stale: mirror push REFUSED

Baseline = 101 curator-triaged false-positive lines (Luhn-timestamp filenames, Epic registry IDs, office phones, codename digits).
